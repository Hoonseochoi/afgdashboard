# page.tsx 한글 깨짐 원인 분석

## 요약

- **조치**: `src/app/page.tsx`를 Git HEAD(배포된 마지막 커밋) 버전으로 **롤백** 완료. 빌드 정상 통과.
- **깨짐 규모**: 수정본에는 한글/닫는태그 등 **600곳 이상**에서 `?`·`?/span>` 등으로 손상됨.

---

## 깨짐이 발생한 이유 (추정)

### 1. **인코딩 불일치 (가장 유력)**

- 소스는 **UTF-8**로 저장돼 있어야 한글이 안전합니다.
- 아래와 같은 경우 UTF-8이 깨질 수 있습니다.
  - **다른 인코딩으로 저장**: 에디터에서 "다른 이름으로 저장" 또는 "인코딩으로 저장" 시 **UTF-8이 아닌** CP949(한글 Windows 기본), EUC-KR 등으로 저장한 경우.
  - **툴/스크립트로 덮어쓰기**: PowerShell, CMD, 배치 등에서 파일을 생성·수정할 때 **기본 코드페이지(CP949)** 로 출력하면, UTF-8이었던 한글 바이트가 잘못 해석되거나 `?`로 치환됩니다.
  - **복사·붙여넣기**: 다른 인코딩(예: CP949) 문서에서 복사한 한글을 UTF-8 파일에 붙여넣을 때 에디터가 잘못 해석하면 깨짐이 납니다.

### 2. **PowerShell/터미널 파이프**

- `Get-Content`, `Set-Content`, `Out-File` 등은 **기본 인코딩**을 쓰는데, Windows PowerShell 기본값이 UTF-8이 아닐 수 있습니다.
- 예: `Get-Content page.tsx | Set-Content page_fixed.tsx`  
  → UTF-8이 아닌 인코딩으로 다시 쓰이면서 한글이 `?` 또는 깨진 문자로 바뀔 수 있습니다.
- 터미널에서 "한글 깨짐 수정" 목적으로 파일을 읽고 다시 쓴 경우, 위와 같은 경로로 **한 번에 대량 깨짐**이 날 수 있습니다.

### 3. **닫는 태그가 `?/span>`, `?/p>` 등으로 바뀐 이유**

- 원래 `</span>`, `</p>` 등은 `<` + `/` + `tagname` + `>` 입니다.
- 인코딩 변환 시 **`<`(U+003C)** 가 일부 환경에서 다른 문자(예: 한글 조합용 바이트)로 해석되거나, 그 주변 한글과 합쳐지면서 **`<`가 사라지고 `?`만 남는** 형태로 깨진 것으로 보입니다.
- 그래서 "문자열은 어느 정도 보이는데 닫는 태그만 `?/span>`처럼 보인다" 같은 현상이 나옵니다.

### 4. **정리**

| 원인 유형           | 설명 |
|--------------------|------|
| **저장 인코딩**    | UTF-8이 아닌 인코딩(CP949 등)으로 저장해 한글 바이트가 깨짐. |
| **쉘/스크립트 처리** | 파이프·리다이렉트로 파일을 읽고 쓸 때 기본 코드페이지로 인해 UTF-8이 깨짐. |
| **복사·붙여넣기**  | 다른 인코딩 문서에서 온 한글이 UTF-8 파일에서 잘못 해석됨. |

---

## 앞으로 깨짐 방지 방법

1. **항상 UTF-8로 저장**
   - VS Code/Cursor: 하단 상태바에서 인코딩 확인 → "UTF-8"로 통일. 필요 시 "Save with Encoding" → "UTF-8" 선택.
2. **PowerShell에서 파일 쓸 때**
   - `Set-Content -Encoding UTF8` 또는 `Out-File -Encoding utf8` 사용.
   - 한글 포함 파일은 가급적 **에디터에서만** 수정하고, 터미널로 "전체 덮어쓰기"는 피하기.
3. **Git**
   - 커밋 전에 `page.tsx` 등 한글 파일을 열어 한두 군데라도 한글이 정상인지 확인 후 커밋.

---

## 이번에 한 작업

- `git checkout HEAD -- afg-dashboard/src/app/page.tsx` 로 **배포된 마지막 커밋 버전**으로 복원.
- 복원 후 `npm run build` 로 빌드 성공 확인.
- 이후 **1월 파트너 시상·4·5·6·9·10번 로직** 등은 이 복원된 파일을 기준으로 **다시 적용**하면 됩니다. 적용 시 위 인코딩 규칙만 지키면 됩니다.
