# 파트너 시상 대시보드 플랜 (리콜 기준)

> 리콜 사진(1월 파트너 시상 화면)을 기준으로 정리한 **새 파트너 시상 대시보드** UI·데이터·구현 방향입니다.

---

## 1. 리콜에서 파악한 화면 구조

### 1.1 전체 레이아웃

- **상단**: meritz 로고, 개발자 관리자/로그아웃, 내보내기, 사용자명(지사명 포함)
- **섹션**: 파트너 시상 카드 그리드
  - **1월**: 11개 카드 — 상단 4열 → 중간 4열 → 하단 2열 → 마지막 1열(메리츠클럽+)
  - **2월**: 9개 카드 — 3열 그리드 유지
- **하단**: 면책 문구 + 업데이트 날짜(예: 02.27)

### 1.2 카드 공통 요소

| 요소 | 설명 |
|------|------|
| **번호 + 제목** | 예: "1. 정규+파트너 추가", "2. 1주차 인보험" |
| **실적 문구** | 카드별 실적 요약 (예: "1주차 인보험 실적 15만", "12월 구간(BJ) 70만 · 1월 구간(BK) 42만") |
| **회차/비율** | "13회차 100%" 또는 "13회차 최대300%" 등 |
| **구간 버튼** | 10만 / 20만 / 30만 / 50만 — **달성한 구간만 강조(선택 상태)** |
| **예상 시상금** | 카드 하단 굵은 주황색, 단위 만원 (0만원 포함) |
| **배경색** | 카테고리별 구분(녹색·하늘색·보라색·노란색) |

### 1.3 카드별 상세 (1월 11개)

| 번호 | 제목 | 실적 문구 예시 | 회차/비율 | 구간 버튼 | 예상 시상금 소스 |
|------|------|----------------|-----------|-----------|------------------|
| 1 | 정규+파트너 추가 | 1월 실적 56만 | 450% 익월 | 없음 | 1월 실적×450% 또는 엑셀 시상금 |
| 2 | 1주차 인보험 | 1주차 인보험 실적 15만 | 13회차 100% | 10/20/30/50만 | `productWeek1PrizeJan` 또는 티어 계산 |
| 3 | 1주차 상품 | 1주차 상품 실적 15만, 통합·간편·어린이 | 13회차 100% | 10/20/30/50만 | 동일(1주차 시상금) |
| 4 | 2주차 인보험 | 2주차 인보험 실적 20만 | 13회차 100% | 10/20/30/50만 | `productWeek2PrizeJan` 또는 티어 |
| 5 | 2주차 상품 | 2주차 상품 실적 20만, 통합·간편·어린이 | 13회차 100% | 10/20/30/50만 | 동일(2주차 시상금) |
| 6 | 12~1월 연속가동 | 12월 구간(BJ) 70만 · 1월 구간(BK) 42만 | 13회차 최대300% | 10/20/30/50만 | `continuous121Prize` |
| 7 | 3주차 인보험 | 3주차 인보험 실적 7만 | 13회차 100% | 10/20/30/50만 | `week3PrizeJan` |
| 8 | 4주차 인보험 | 4주차 인보험 실적 0만 | 13회차 100% | 10/20/30/50만 | `week4PrizeJan` |
| 9 | 1~2월 연속가동 | 1-2월 연속가동 실적 14만 | 13회차 최대300% | (연속가동 구간 표시) | `continuous12Prize` |
| 10 | 1~2월 추가 연속가동 | 1-2월 추가 연속가동 실적 7만 | 13회차 최대300% | (연속가동 구간 표시) | `continuous12ExtraPrize` |
| 11 | 메리츠클럽 플러스 | 1월 달성 구간 목표·2월 실적, 2월 94만/목표 40만 | 13회차, 조건 충족 | 진행 바(완성) | `meritzClubPlusPrize` |

### 1.4 카드 색상 구분 (리콜 기준)

| 색상 | 적용 카드 |
|------|-----------|
| **녹색** | 1(정규+파트너), 3(1주 상품), 5(2주 상품), 8(4주 인보험) |
| **하늘색** | 2(1주 인보험), 4(2주 인보험), 7(3주 인보험) |
| **보라색** | 6(12~1월 연속가동), 9(1~2월 연속가동), 10(1~2월 추가 연속가동) |
| **노란색** | 11(메리츠클럽 플러스) |

### 1.5 구간 버튼(10/20/30/50만) 로직

- **인보험/상품 주차 카드(2~5, 7~8)**: 실적이 10만 이상이면 10만 강조, 20만 이상이면 10·20만 강조 … (해당 티어까지 선택)
- **연속가동 카드(6, 9, 10)**: 12월·1월(또는 1월·2월) 구간이 각각 10/20/30/50만을 만족하면 해당 버튼 강조. 6번은 "12월 구간 70만 · 1월 구간 42만"처럼 두 구간 모두 표시.

## 3. 데이터 요구사항

### 3.1 이미 있는 데이터

- `selectedAgent.performance`, `weekly`, `partner` (파싱된 `PartnerPrizeData`)
- 1월: `januaryClosed[code]` 또는 `weekly`/`performance` 로 주차·월간 실적
- 12~1월 연속가동: `p.continuous121Dec`, `p.continuous121Jan`, `p.continuous121Prize`
- 1~2월 연속가동: `p.continuous12Jan`, `p.continuous12Feb`, `p.continuous12Prize` 등

### 3.2 카드별로 필요한 값

- **실적 문구**: 주차 실적(`viewW1`, `viewW2`, `viewW3`), 4주차 실적, 12월/1월 구간, 1월/2월 구간 등
- **달성 구간**: 실적 → 10/20/30/50만 중 어디까지 달성했는지 (티어 배열과 비교)
- **예상 시상금**: 기존 `p.*` 필드 및 `getPartnerTierPrize` 활용

## 4. 구현 플랜 (작업 순서)

### Phase A: 카드 컴포넌트 확장

- [ ] **A.1** `PartnerPrizeCard` 를 **확장**하거나, 파트너 전용 **새 카드 컴포넌트** 도입  
  - props: `index`, `title`, `performanceLabel`, `periodLabel`(13회차 100% 등), `tierPerf`(실적 또는 구간별 실적), `expectedPrize`, `variant`(색상: green | sky | purple | yellow)
- [ ] **A.2** 10/20/30/50만 구간 버튼 영역 구현  
  - 실적 → 달성 티어 계산 (예: `[10,20,30,50].filter(t => perf >= t*10000)`)  
  - 달성한 구간만 강조 스타일
- [ ] **A.3** 연속가동 카드(6, 9, 10)는 구간이 2개인 경우 문구 2줄 표시 (예: "12월 구간(BJ) 70만 · 1월 구간(BK) 42만")

### Phase B: 1월 11카드 데이터 매핑

- [ ] **B.1** 1~11번 각 카드에 넣을 `performanceLabel`, `periodLabel`, `tierPerf`, `expectedPrize` 계산 로직 정리  
  - 1번: 1월 실적, 450% 익월, 예상 시상금(정규+파트너)  
  - 2~5, 7~8: 주차별 실적 + 티어 강조 + 시상금  
  - 6: continuous121Dec / continuous121Jan + continuous121Prize  
  - 9~10: continuous12* / continuous12Extra* 실적 문구 + 시상금  
  - 11: 메리츠클럽+ 목표/2월 실적 + 진행 바
- [ ] **B.2** 1월용 `variant` 배치: 1·3·5·8 녹색, 2·4·7 하늘색, 6·9·10 보라색, 11 노란색

### Phase C: 2월 9카드 적용

- [ ] **C.1** 2월용 카드에도 동일한 확장 카드 사용  
  - 실적 문구: viewW1, viewW2, week34Sum, continuous12 구간 등  
  - 구간 버튼은 2월 규칙에 맞게 적용
- [ ] **C.2** 2월 그리드: `md:grid-cols-3` 유지, 색상은 1월과 유사하게 매핑

### Phase D: 메리츠클럽+ 카드

- [ ] **D.1** 11번 카드만 진행 바(progress bar) 표시  
  - "1월 달성 구간 목표·2월 실적", "2월 94만 / 목표 40만"  
  - 조건 충족 시 완성 스타일(예: 100% 채워진 바)
- [ ] **D.2** 목표 구간·현재 실적은 기존 `meritzClubPlusPrize` 계산 로직과 연동

### Phase E: 그리드·반응형

- [ ] **E.1** 1월: 상단 4열 → 2열(중간) → 1열(메리츠클럽+)  
  - Tailwind: `grid-cols-1 sm:grid-cols-2 md:grid-cols-4`, 마지막 카드만 `md:col-span-4` 또는 전용 행
- [ ] **E.2** 2월: `md:grid-cols-3` 유지

### Phase F: 푸터

- [ ] **F.1** "상기 시상내용은 예상값이며, 참고용으로만 활용하시기 바랍니다."
- [ ] **F.2** "업데이트 날짜 : MM.DD" — `updateDate` 또는 데이터 기준일 표시

## 5. 현재 코드와의 차이 요약

| 항목 | 현재 구현 | 리콜 기준 목표 |
|------|-----------|----------------|
| 카드 내용 | 제목 + 예상 시상금만 | 번호 + 제목 + 실적 문구 + 회차/비율 + 구간 버튼(10/20/30/50) + 예상 시상금 |
| 구간 표시 | 없음 | 달성한 10/20/30/50만 구간 강조 |
| 연속가동 | 제목+금액만 | 12월·1월(또는 1·2월) 구간 실적 문구 2줄 |
| 메리츠클럽+ | 금액만 | 목표/실적 문구 + 진행 바 |
| 색상 | 통일 스타일 | 카드 유형별 녹색/하늘색/보라색/노란색 |
