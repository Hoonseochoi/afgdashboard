# 데일리 MC_LIST 엑셀 매칭 분석

`supabase-upload-daily.js`가 엑셀에서 어떻게 컬럼을 찾고, Supabase에 어떻게 반영하는지 정리한 문서입니다.

---

## 1. 엑셀 컬럼 매칭

엑셀 열: A=1, B=2, C=3, D=4, E=5, **F=6**, G=7, … **K=11**, … (1부터 셈).  
스크립트는 배열 인덱스(0부터)를 쓰므로 F→5, K→10 입니다.

| 엑셀 열 | 열(1부터) | 인덱스(0부터) | 용도 | 검색 키워드 |
|--------|-----------|---------------|------|-------------|
| F | 6 | 5 | 코드(사번) | `사용인코드`, `코드` |
| G | 7 | 6 | 설계사명 | `설계사명`, `이름` |
| **K** | **11** | **10** | **인정실적(당월)** | `인정실적`, `인정 실적`, `당월실적` |
| S | 19 | 18 | 전월실적 | `전월실적` |
| U~X | 21~24 | 20~23 | 1~4주차 | `1주차`~`4주차` |
| AL | 38 | 37 | 지사명 | `지사명` |

- **F열 = 6번째 열** (코드), **K열 = 11번째 열** (인정실적). 코드에서는 0-based로 F→5, K→10.
- 3월 파일(`*_202603.xlsx`)인 경우 **항상 K열(10)을 당월 인정실적으로 사용**하도록 스크립트에서 강제합니다.

---

## 2. 헤더 행 찾기

- **처음 15행** 안에서 다음이 **포함된 셀**이 있는 **첫 번째 행**을 헤더로 사용합니다.
  - `인정실적` 또는 `당월실적` 또는 `사용인코드`
- 해당 행의 각 셀 문자열을 위 키워드로 검색해 컬럼 인덱스를 정합니다.
- 헤더를 못 찾으면 **0행**을 헤더로 쓰고, 기본 인덱스(위 표)로 읽습니다.

---

## 3. K열(인정실적)이 반영되지 않는 경우 점검

### 3.1 3월 파일인데 반영 안 될 때

- 스크립트는 **3월(currentMonthKey === '2026-03')** 이면 **무조건 인덱스 10(K열)** 을 당월 인정실적으로 사용합니다.
- 실행 시 로그에 다음이 나옵니다:
  - `[파싱] 헤더 행: N | ... 인정실적(당월): 10 (K열=10) ... | K열 헤더: (실제 K열 헤더 텍스트)`
- **K열 헤더**가 비어 있거나 다른 값이어도, **데이터는 10번 컬럼**에서 읽습니다.

### 3.2 값이 0으로만 나올 때

- K열 셀 타입: 엑셀에서 **숫자**면 그대로, **문자**면 `,`·공백 제거 후 숫자로 변환합니다.
- **빈 셀**이면 0으로 들어갑니다.
- **날짜/문자**로 저장돼 있으면 숫자 변환 실패 시 0이 됩니다. → 엑셀에서 K열이 **숫자** 형식인지 확인하세요.

### 3.3 행이 아예 빠질 때

- **지사명**에 `어센틱`이 **포함**된 행만 처리합니다.
- **코드**가 비어 있거나 **길이 5 미만**이면 해당 행은 제외됩니다.

---

## 4. Supabase 반영 흐름

1. **엑셀 파싱**  
   `parseDailyXlsx(filePath, { currentMonthKey, logColumns: true })`  
   - 3월이면 당월 실적 = **항상 K열(인덱스 10)**  
   - 로그에 헤더 행, 인정실적 컬럼 인덱스, K열 헤더가 찍힘.

2. **기존 agents 전량 조회**  
   `fetchAllAgentsMap()`  
   - DB의 모든 설계사를 페이지 단위로 읽고,  
   - `normalizeCode(code)`로 키 통일 후 맵 생성.

3. **코드 정규화**  
   - `normalizeCode(c)`: 숫자로 해석 가능하면 앞자리 0 제거한 문자열로 통일.  
   - 엑셀 `0318001118` ↔ DB `318001118` 매칭 가능.

4. **업데이트/삽입**  
   - 엑셀 행의 코드가 맵에 있으면 → `performance[당월키]`, `weekly` 만 **update**.  
   - 없으면 → **insert** (신규).

5. **agent-order.json**  
   - MC_LIST에 나온 순서(중복 제거) + **DB에만 있는 코드**를 뒤에 붙여 저장.  
   - **리스트 정렬**은 API에서 **3월(2026-03) 매출 내림차순**으로 하므로, 이 파일 순서와는 무관합니다.

---

## 5. 배치가 안 되는 경우 체크리스트

| 항목 | 확인 |
|------|------|
| 파일 위치 | `data/daily/` 아래에 `NNNNMC_LIST*.xlsx` (예: `0305MC_LIST_OUT_202603.xlsx`) |
| 사용 파일 | 같은 폴더 내 **파일명 정렬 시 마지막** 파일이 선택됨 |
| 3월 파일 | 파일명에 `_202603.xlsx` → 당월 실적은 **반드시 K열(10)** 사용 |
| 로그 | `[파싱] 헤더 행: ... 인정실적(당월): 10 ...` 확인 |
| 디버그 코드 | 코드 `725070184` 등 지정 시 `[디버그] 725070184 엑셀 인정실적(당월): ??? 원` 로그로 엑셀에서 읽은 값 확인 |
| Supabase | `.env.local`의 `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`가 실제 사용 중인 프로젝트와 동일한지 확인 |

---

## 6. 리스트 순서 (대시보드/API)

- **항상 3월(2026-03) 매출 순**으로 정렬됩니다.
- `performance['2026-03']` 값이 **큰 순** → 같으면 코드 오름차순.
- `agent-order.json`은 “MC_LIST 순서 + DB전용”만 저장하고, **화면 정렬에는 사용하지 않습니다.**

---

## 7. PRIZE_SUM (1주차 상품) 엑셀

- **파일 패턴**: `data/daily/` 아래 `NNNNPRIZE_SUM*.xlsx` (예: `0305PRIZE_SUM_OUT_202603.xlsx`).
- **스크립트**: `supabase-upload-march-product-week1.js` (데일리 업데이트 시 `run-daily-update.py`에서 MC_LIST 후 자동 실행).
- **고정 인덱스** (0-based): K열(10)=설계사코드, AC열(28)=상품 1주차 실적.
- **Supabase 반영**: `agents.product_week1`, `agents.weekly.productWeek1`.
