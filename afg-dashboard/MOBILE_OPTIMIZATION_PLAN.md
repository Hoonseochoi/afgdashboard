# 모바일 UI 최적화 플랜

현재 대시보드(`page.tsx`)와 로그인(`login/page.tsx`) 구조를 기준으로, 모바일에서 UI가 깨지는 부분을 정리하고 수정 순서를 제안합니다.

---

## 1. 현황 요약

- **헤더**: 로고 + 검색창(고정폭) + 리스트 버튼 + 유저정보가 한 줄 → 좁은 화면에서 넘침/겹침
- **메인 상단**: 프로필 + 예상 시상금 + 누적 실적 카드에 `min-w-[200px]` 등 → 가로 스크롤 또는 찌그러짐
- **고정 하단 바**: 면책문구가 길어서 한 줄에 넣으면 모바일에서 줄바꿈/겹침
- **차트/그리드**: 구조는 `grid-cols-1`로 모바일 1열이지만, 차트 높이·패딩·폰트는 추가 조정 필요

---

## 2. 수정 우선순위 및 작업 목록

### Phase 1: 헤더 (모바일에서 가장 먼저 깨지는 부분)

| 순서 | 작업 | 설명 |
|------|------|------|
| 1-1 | 헤더 레이아웃 반응형 | 모바일에서 2줄로 분리: 1줄=로고+로그아웃, 2줄=검색+리스트 버튼(풀너비 또는 flex-1). 또는 768px 미만에서 검색창을 아이콘만 보이게 하고 탭 시 풀너비 검색으로 전환 |
| 1-2 | 검색창 너비 | `w-64` → 모바일에서 `w-full max-w-[200px]` 또는 상단 풀너비로 변경해 넘침 방지 |
| 1-3 | 로고/브랜드 텍스트 | 모바일에서 `text-xl` 등으로 축소해 한 줄 유지 |

### Phase 2: 메인 콘텐츠 (프로필 + 시상 카드)

| 순서 | 작업 | 설명 |
|------|------|------|
| 2-1 | 상단 카드 영역 | `min-w-[200px]` 제거 또는 모바일에서 `min-w-0` 적용. `flex-1`만으로 너비 결정되도록 해 가로 스크롤 방지 |
| 2-2 | 프로필 영역 패딩/폰트 | 모바일 `p-4`, 제목 `text-2xl` 등으로 축소해 세로 공간 절약 |
| 2-3 | "나의 시상 현황" 그리드 | 이미 `grid-cols-1`이므로 유지. 카드 내부 `p-4` 유지 또는 모바일 `p-3` |
| 2-4 | 2월 목표 + 차트 영역 | 모바일에서 차트 `h-56` 또는 `min-h-[200px]`로 높이 조정, 터치 시 툴팁이 가리지 않도록 여백 확인 |

### Phase 3: 고정 하단 바

| 순서 | 작업 | 설명 |
|------|------|------|
| 3-1 | 면책문구 + 업데이트 날짜 | 모바일에서 **두 줄** 배치: 1줄=면책문구(작은 폰트), 2줄=업데이트 날짜. 또는 `flex-col items-center gap-1` + `text-center` |
| 3-2 | 폰트/패딩 | `text-[10px]` 또는 `text-xs`, `py-2 px-3` 등으로 줄여서 하단 바 높이 축소 |

### Phase 4: 전역·접근성

| 순서 | 작업 | 설명 |
|------|------|------|
| 4-1 | viewport / 탭 영역 | `layout.tsx` 또는 `_document`에 viewport meta 확인. 버튼/링크는 최소 44px 터치 영역 유지 |
| 4-2 | 로그인 페이지 | 이미 `sm:max-w-md` 등 반응형 있음. 모바일에서 입력창·버튼 패딩만 점검 |
| 4-3 | Recharts 터치 | 모바일에서 차트 드래그/줌 비활성화 또는 터치 스크롤과 충돌하지 않도록 옵션 확인 |

---

## 3. 권장 Tailwind 브레이크포인트 사용

- `sm:` 640px 이상
- `md:` 768px 이상
- `lg:` 1024px 이상

모바일 우선으로 작성 후 `sm:`, `md:`, `lg:`로 점진적 개선하는 방식이 유리합니다.

---

## 4. 적용 순서 제안

1. **Phase 1 (헤더)** → 화면 최상단이 정리되면 나머지도 보기 쉬움  
2. **Phase 3 (고정 하단 바)** → 짧은 수정으로 모바일에서 바로 체감  
3. **Phase 2 (메인)** → 카드·차트 세부 조정  
4. **Phase 4** → viewport·접근성·로그인 마무리  

원하시면 Phase 1부터 실제 `page.tsx` 수정 예시(클래스명·구조)까지 구체 코드로 작성해 드리겠습니다.
